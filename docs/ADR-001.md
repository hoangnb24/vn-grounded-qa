# ADR-001 — Architecture Direction

**Status:** Approved  
**Date:** 2026-03-09  
**Context:** Vietnamese grounded QA for controlled enterprise document corpora

---

## Decision

Build around **structured sparse retrieval + bounded tool orchestration**, not around vector-DB-centric RAG, open-ended agents, or GraphRAG.

The architecture is:

1. Parser-neutral ingestion into an intermediate representation
2. Canonical storage at document-unit level (not whole files)
3. Sparse search + metadata filtering as retrieval backbone
4. Bounded semantic tools for search, read, expansion, term resolution
5. Grounded answer synthesis with citations and no-answer policy
6. Evidence graph / version graph as upgrade paths, not day-one dependencies

## Why this architecture

It balances six needs simultaneously:

| Need | How it's addressed |
|---|---|
| Accuracy | Evidence-unit retrieval, not chunk-level approximation |
| Traceability | Every answer traces to source documents and units |
| Vietnamese handling | Segmentation, normalization, alias fields built into index |
| Mixed document support | Parser-neutral IR handles legal, SOP, Markdown, tables, FAQ |
| Operational simplicity | SQLite FTS5, no vector DB, no GPU infra required |
| Upgrade flexibility | Clear paths to reranker, hybrid retrieval, graph layer |

## Why sparse retrieval as default

- Fits the "no mandatory vector DB" constraint
- Keeps search explainable and debuggable
- Works naturally with structural metadata (headings, doc types, versions)
- Allows incremental adoption of neural layers later
- [Keyword Search Is All You Need](https://arxiv.org/abs/2602.23368) supports sparse-first for controlled corpora
- [SQLite FTS5](https://sqlite.org/fts5.html) provides mature ranking, tokenizer options, and fielded search

This is a validated starting point, not a permanent position. The plan explicitly includes M5 (thin RAG baseline comparison) to test whether this bet holds.

## Why Vietnamese changes the design

- **Segmentation**: Vietnamese word boundaries are not reliably whitespace-delimited. [VnCoreNLP](https://aclanthology.org/N18-5012/) segmentation is required for search quality.
- **Mixed-language queries**: Enterprise users mix Vietnamese with English acronyms, UI labels, product names, regulation codes. This requires alias dictionaries, ASCII folding, and identifier-aware search as first-class fields.
- **Domain terminology**: Legal/policy/technical corpora are terminology-sensitive. Term resolution is part of retrieval quality, not an add-on.

## Rejected starting points

| Approach | Why rejected |
|---|---|
| Vector-DB-first RAG | Increases infra scope before proving sparse is insufficient |
| Open-ended agent loops | Tool-use benchmarks ([BFCL](https://gorilla.cs.berkeley.edu/leaderboard), [MCPMark](https://huggingface.co/papers/2509.24002)) show multi-step execution remains brittle for production |
| GraphRAG from day one | Only justified after observing graph-shaped failures (reference chains, version inheritance) |
| Full knowledge graph | High modeling cost before proving the basic retrieval boundary |
| Whole-corpus stuffing | Doesn't scale across heterogeneous corpora; [LaRA benchmark](https://researchportal.hkust.edu.hk/en/publications/lara-benchmarking-retrieval-augmented-generation-and-long-context/) shows no universal winner |

These are deferred, not forbidden. Each has a defined trigger for re-evaluation.

## Upgrade path (in order)

1. Terminology / alias enrichment
2. Query rewrite
3. Local reranker
4. Evidence graph
5. Version graph
6. Hybrid neural retrieval (only if sparse recall stays insufficient after steps 1–5)

## Decision triggers for re-evaluation

- Sparse recall stays below threshold after tuning, terminology, segmentation, rewrite, and reranking
- Multi-document joins fail primarily due to inadequate relation following
- Legal/compliance domains need stronger version reasoning than metadata overlays
- Parser-neutral IR proves too lossy for a strategic document type
- Latency/cost becomes worse under bounded orchestration than a thinner hybrid approach
